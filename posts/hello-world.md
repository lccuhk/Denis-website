# 初入江湖，代码为剑

> 说出来你可能不信，我第一次写代码的时候，连 `Hello, World!` 都跑不起来。

## 那是一个普通的下午

记得那是大二上学期，计算机导论课。老师在上面讲得唾沫横飞，我在下面昏昏欲睡。

"今天我们来写第一个程序，输出 Hello, World!"

我迷迷糊糊地打开电脑，照着课本敲了几行代码，按下运行——

```
错误：语法错误
```

嗯？不对啊，我明明照着敲的。再检查一遍——哦，少打了个分号。

加上分号，再运行——

```
错误：未定义的符号
```

……行吧，大小写搞错了。

就这么折腾了快半小时，屏幕上终于出现了那行字：

```
Hello, World!
```

你猜怎么着？我居然有点小激动。

就像你花了半天时间，终于把一个复杂的机关盒子打开了，虽然里面只是一张写着"恭喜你"的小纸条，但那种成就感，真的会上瘾。

## 为什么是编程？

有人问我，好好的学什么不好，非要学编程？掉头发不说，还容易猝死。

我说，你不懂。

编程这东西，有种魔力：

- **你说的算**：电脑是个死脑筋，你说往东它绝不往西。错了也是你错了，它不跟你抬杠。
- **从无到有**：几行代码，就能变出一个网站、一个游戏、一个能帮你干活的工具。这种创造的快感，别的东西给不了。
- **永无止境**：今天你觉得自己很牛了，明天就能看到一个更牛的。学不完，真的学不完，但这种"永远有新东西"的感觉，让人着迷。

说白了，编程就像练武。你练的是内功（算法、数据结构），耍的是招式（编程语言、框架），行走江湖靠的是经验（踩过的坑）。

## 我的"修炼"之路

说出来不怕你笑，我这一路走来，坑踩得比路多。

### 第一阶段：以为自己很行

刚学完 C 语言，觉得自己天下无敌了。不就是写代码吗？有什么难的？

然后就被数据结构按在地上摩擦。

链表？指针？二叉树？

我至今记得那个通宵，就为了搞懂"递归"是怎么回事。第二天早上照镜子，眼睛里都是红血丝，但是脑子里突然"叮"的一声——哦，原来如此！

那种感觉，就像你练了半年的武功，突然打通了任督二脉。

### 第二阶段：发现自己不行

后来学前端，学后端，学移动端。

越学越发现——我怎么什么都不会？

今天出个 React，明天来个 Vue，后天又蹦出个 Svelte。刚把这个搞明白，那个又更新了。

焦虑过，迷茫过，甚至想过放弃。

后来一位前辈跟我说：

> "技术这东西，追是追不完的。练好内功，招式只是顺手拈来。"

我恍然大悟。

从此不再盲目追新，而是沉下心来打基础。计算机网络、操作系统、编译原理……这些"没用"的东西，反而成了我后来最有用的东西。

### 第三阶段：接受自己不行

现在我终于想通了——

没有人什么都会。

你不需要成为全栈工程师，你只需要成为某个领域的专家。

你不需要记住所有 API，你只需要知道去哪里查。

你不需要写出完美的代码，你只需要写出能跑、能维护、能迭代的代码。

接受自己的不完美，反而能走得更远。

## 给后来人的几句心里话

如果你刚入门，或者正在犹豫要不要入行，我想跟你说几句掏心窝子的话：

**第一，别怕错。**

Bug 是最好的老师。我现在看到报错，第一反应不是慌，而是兴奋——又有东西可以学了。

**第二，多动手。**

看十遍教程，不如自己写一遍。写十遍 Hello World，不如写一个真正能用的小工具。

**第三，别熬夜。**

真的，头发掉了就长不回来了。身体是革命的本钱，这句话等你到我这个年纪就懂了。

**第四，多交流。**

别一个人闷头学。加入社区，写写博客，跟同行聊聊。你会发现，原来大家都一样菜，只是人家不说而已。

**第五，保持好奇。**

技术更新很快，但底层的东西没变。保持对新技术的好奇心，但别被新技术牵着鼻子走。

## 最后

有人说，程序员是吃青春饭的。

我不这么认为。

编程这行，经验真的很重要。你踩过的坑、解决过的问题、设计过的系统，都是你的财富。

只要你保持学习，保持热情，这碗饭，你能吃到老。

当然，前提是你注意身体。

好了，不说了，改 Bug 去了。

江湖路远，我们后会有期。
